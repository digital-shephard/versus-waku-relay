# Community nodes

A compatible community node may transport Versus traffic without becoming a protocol authority. It must use the coordinated cluster and shard configuration, support WSS LightPush/Filter/Store as advertised, bound retention, publish stable multiaddresses, and disclose operator availability assumptions.

Community addresses are owner-selectable additions. Inclusion in a default desktop bootstrap list requires public health evidence, version disclosure, bounded retention, abuse policy, and an accepted compatibility run. Wealth, sponsorship, or message volume does not buy default-bootstrap placement.
